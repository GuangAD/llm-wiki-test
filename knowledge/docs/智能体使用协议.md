# 智能体使用协议

## 意图映射

| 用户意图 | 命令 |
| --- | --- |
| 保存链接、文件、文本、想法 | `uv run kb ingest <input>` |
| 盘点知识库内容、数量、标签、来源 | `uv run kb status` |
| 查询具体主题、观点、概念 | `uv run kb ask "<question>"` |
| 继续主题页编译 | `uv run kb compile --continue <job_id> --topic-key <topic_key>` |
| 生成可写选题 | `uv run kb brief topics` |
| 生成知识周报 | `uv run kb brief weekly` |

## 盘点类问题

以下问题属于盘点类意图：

- 当前知识库中有哪些内容
- 当前有多少笔记
- 有哪些标签
- 有哪些来源
- 最近入库了什么

处理规则：

1. 优先执行 `uv run kb status`。
2. 根据命令返回的 `counts / recent_notes / tags / sources` 组织回复。
3. 不使用 `rg`、`Get-ChildItem` 等方式全局搜索 `knowledge/` 来盘点内容。
4. 如果 `kb status` 不可用，只能只读 `indexes/recent.md`、`indexes/tags.md`、`indexes/sources.md` 兜底。
5. 只有索引缺失或明显损坏时，才允许只读检查 `notes/**/*.md`，并在回复中说明兜底原因。

## 两段式生成流程

1. 执行命令，得到 `needs_generation`。
2. 读取返回的 `generation_request_path`。
3. 按请求读取来源文件和 prompt。
4. 写入 `generation_result_path`。
5. 执行对应 `--continue` 命令。

## 回复规则

- 回复用户时说明实际执行的 `kb` 命令。
- 查询或生成内容时必须附来源文件或来源链接。
- 失败时只返回命令给出的错误码、错误信息和下一步动作，不编造结果。

## 边界

- 本目录就是知识库根目录，命令默认以当前目录为知识库根。
- 智能体不要把外层 `docs/`、`tests/` 或工程配置当作知识库内容。
- 智能体不要直接编辑 `raw/`、`notes/`、`wiki/`、`briefs/`、`indexes/`、`state/`，除非用户明确要求修复数据。
